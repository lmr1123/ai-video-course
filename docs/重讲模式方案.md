# AI 教师重讲模式方案

> 建立日期：2026-07-12。对应 PRD 4.7 的"AI 教师重讲模式"。全文标注验证状态：**纯推演 / 待验证 / 已验证**。前置依赖：黄金测试样例课程（已有 1 条）。

## 1. 定位与边界

状态：**已定**（继承既有决策）。

重讲不是按原视频顺序缩写，而是像优秀技术博主一样重建叙事：先制造问题，再给模型，用案例验证，落到项目决策（叙事骨架见课程编排文档 4.2，此处不重复）。边界：

- 不是数字人朗读摘要；镜头单位是"一个教学意图"，不是一段文字。
- 不复用原片画面与音频（版权决策：重讲版只用自产素材）；引用原视频只以"时间戳跳转链接"形式出现。
- 重讲的输入是课程中间产物（导读、知识地图、深讲模块、案例），不重新理解原视频（成本决策）。

## 2. 产物链

状态：**已验证**（2026-07-12，首集跑通：17 镜脚本 → edge-tts 旁白 5.8 分钟 → 网页合成课 `prototype/relecture/` → ffmpeg 与 HyperFrames 双路 MP4 导出，见 `tasks/progress.md`）。

```text
课程中间产物（已有）
→ 重讲脚本（叙事重组，强模型一次）
→ 分镜表（脚本 → 镜头序列，schema 见 §3）
→ L1 音频课（TTS，10–30 分钟）
→ L2 网页合成课（分镜页面 + 音频同步）
→ L3 mp4 导出（后置，验证叙事质量后才做）
```

关键决策：**L2 网页合成课是重讲模式的主形态**，而非真视频渲染。

- 分镜用 HTML/SVG 渲染（机制图、对比卡、代码聚焦、案例白板——即编排文档 3.4 的镜头类型），音频播放时按时间轴切换分镜并高亮当前句。
- 保留课程页的全部交互：暂停追问、点时间戳回原片、跳章节、变速。真 mp4 一旦渲染这些全部丢失。
- 复用互动课程页原型的技术栈（播放同步逻辑已验证），把"YouTube 播放器"换成"音频 + 分镜画布"。
- mp4 导出（L3）选型 **HyperFrames**（HeyGen 开源的 HTML→MP4 确定性渲染引擎，Apache 2.0，Puppeteer+FFmpeg，agent-native）：我们的分镜本来就是 HTML/CSS，加渲染层即可导出，不引入新内容管线——所以可以无痛后置。

### 2.1 三条候选路线与选型

状态：**已选型，待实验确认**。重讲模式有三条可行实现路线：

| | 路线 A：NotebookLM 视频概览 | 路线 B：自建管线（脚本＋TTS＋分镜） | 路线 C：HyperFrames 直出 MP4 |
|---|---|---|---|
| 做法 | 上传课程文档＋字幕，托管生成"幻灯片＋旁白"视频 | 强模型写重讲脚本 → TTS → HTML 分镜同步页（L2） | 让 agent 直接写 HTML 动画，渲染成片 |
| 成本 | ≈0，当天可出 | 每课 2 次强模型＋1 次 TTS | 同 B 还要多一层渲染调试 |
| 叙事可控 | 不可控（不保证钩子→模型→案例→决策） | 完全可控（§3 schema） | 完全可控 |
| 时间戳溯源/归属标注 | 无 | 有（每镜头 source 字段） | 可有，但成片后不可点击 |
| 可交互（暂停追问/跳回原片/变速换深度） | 无 | 全部保留 | 无（MP4 是终态） |
| 与现有课程页融合 | 无法融合，独立成品 | 同一份课程数据直接复用 | 复用 B 的分镜再渲染 |

**选型：B 为主线，A 为基线实验，C 为 B 的导出层——三者不是竞争关系。**

- 选 B 的决定性理由：可溯源＋可交互是本产品的立身之本（PRD 成功指标里"回答能否追溯原视频"），A 和 C 的终态产物都做不到"点一下回到原片 00:33:31 核对"。
- A 的正确用法是基线实验（优先做，成本≈0）：同一份 Codex 课程材料生成视频概览，按测量协议对比复述质量——它直接回答"重讲视频形态值不值得投入"；若它已够用，B 的优先级应下调。本地 `notebooklm-skill` 可自动化上传与问答，视频概览需手动触发。
- C（HyperFrames，HeyGen 开源 HTML→MP4 引擎）不是内容方案而是渲染层：B 的分镜本来就是 HTML，接上它就能导出可分享/离线的 MP4，内容管线只写一次。

### 2.2 与互动课程页的融合设计

状态：**纯推演**。重讲课不是独立产品，是课程页的一种播放模式：

- 课程页顶部增加模式切换：**原片模式**（现状：YouTube 播放器＋知识地图）⇄ **重讲模式**（音频播放器＋分镜画布，同一位置替换）。
- 知识地图两种模式共用：重讲模式下时间轴显示重讲课自身进度，播放同步高亮逻辑直接复用（已验证的 1 秒轮询）。
- 每个分镜的 `source` 时间戳渲染为可点击 chip：点击即切回原片模式并跳到对应位置——"重讲讲不清就回原片"是核心学习动线。
- 重讲脚本的数据源就是页面里已有的深讲模块与知识地图，不新建内容层；换深度/换语言只重生成旁白，分镜复用。

## 3. 重讲脚本与分镜 schema

状态：**纯推演**。

脚本按叙事骨架分"幕"（act），每幕含若干镜头（shot）。镜头是最小单位：

```json
{
  "act": "核心模型",
  "intent": "让学员看见'媒介-阶段绑定断裂'的前后对比",
  "type": "对比卡",           // 旁白卡 | 机制图 | 代码聚焦 | 对比卡 | 案例白板 | 章节检查
  "narration": "过去，看到接近生产环境的东西，就意味着……",
  "visual": { "left": "旧世界：精致度=进度", "right": "新世界：精致度≠进度", "steps": 2 },
  "duration_s": 25,
  "source": ["00:09:04"],     // 可追溯：本镜头对应的原视频时间戳
  "attribution": "归纳"       // 原话|归纳|解释|外部，与课程页同一套标注
}
```

约束：

- 每幕必须有 `source` 覆盖——重讲的每个论断仍可追溯到原片；
- "章节检查"镜头（一道判断题/一句复述提示）每 3–5 分钟一个，暂停等待用户继续；
- 旁白文本即 TTS 输入；`visual.steps` 支持机制图分步出现，与旁白句对齐。

## 4. 生成管线与模型分工

状态：**纯推演**。

| 阶段 | 输入 | 模型/工具 | 次数 |
|---|---|---|---|
| 重讲脚本 | 课程中间产物（约 1–2 万字） | 强模型 | 每课 1 次 |
| 分镜表 | 脚本 | 强模型（同次或二次调用） | 每课 1 次 |
| 分镜渲染 | 分镜 JSON | 确定性模板（HTML/SVG 组件库），不用生成式图像 | 0 token |
| 旁白音频 | narration 文本（10–30 分钟 ≈ 3–8 千字） | 中文 TTS（候选：MiniMax / 豆包 / OpenAI TTS，需试听选型） | 每课 1 次 |
| 机制动画（可选增强） | 个别高价值机制图 | 生成式视频（如 Seedance）| 默认 0，人工挑选 |

成本形态：每课两次强模型调用＋一次 TTS，分镜渲染零边际成本。换语言/换深度只重生成 narration 与 TTS，分镜复用——与"保存中间产物、按需复用"的既有成本设计同构。

## 5. MVP 验证（下一步行动）

状态：**待验证**。

用已完成的 Codex 课程样例手工做一集，不写自动化管线：

1. 手工写 12–15 分钟重讲脚本（四幕：钩子与承诺 → 流程倒转核心模型 → 模型时机案例 → 取舍边界与最小实践＋三句话复盘）；
2. 手工排 20–30 个分镜（用 §3 schema，存 JSON）；
3. TTS 生成音频（顺便完成选型试听）；
4. 在原型页新增"重讲模式"标签页：音频播放器 + 分镜画布 + 同步高亮，复用现有同步逻辑。

测量（对照组：直接阅读互动课程页）：

- 听完重讲后复述两个核心机制，与读课程页的复述质量对比；
- 记录总时长：重讲课时长 vs 阅读用时；
- 主观判断：哪些幕"像课"，哪些幕仍像"念摘要"——逐幕标注，反哺脚本 prompt。

通过标准：重讲版在"不看屏幕也能听懂"场景下成立（通勤测试），且复述质量不低于阅读版。

## 6. 待验证问题

- 叙事重组的脚本 prompt 能否稳定产出"制造问题→给模型→案例→决策"的结构，而不是退化成分模块摘要串烧；
- 章节检查镜头的打断频率多少合适；
- TTS 对中英混排（术语、产品名）的发音质量是否可接受，是否需要发音词典；
- 分镜模板库需要多少种镜头类型才够用（首版按 6 种做）；
- mp4 导出的真实需求有多大——如果网页合成课已满足学习场景，L3 可能永远不做。
