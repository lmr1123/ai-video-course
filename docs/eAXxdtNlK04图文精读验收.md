## 2. 为什么这个样本适合图文

视频虽然是单人技术演讲，但不是纯口播。画面直接承担以下知识：

- “目标天然不完整”的路线示意；
- Agent、目标函数和 Optimizer 的关系；
- fit、validation、test 三组数据的职责；
- 错误分析、更新、验证门槛和停止条件；
- 准确率提升曲线与平台期；
- Optimizer 实际添加的决策规则和例子；
- 低信号连续评分与高信号是/否判断的对照；
- 领域专家、生产数据和泛化机制的落地步骤。

去掉这些画面后，用户仍能听懂结论，但更难建立实验结构、核对数字和把方法迁移到自己的 Agent 评估管线，因此符合“去掉画面后是否明显更难理解、判断或复现”的启用标准。

## 3. Watchless 实跑结果

## 4. 图文样页结构

本地样页将视频重组为三个深讲模块：

1. 循环不是起点，目标函数才是；
2. 用可验证任务观察循环到底学会了什么；
3. 把领域专家的判断变成高信号反馈。

每个模块包含原讲脉络、问题背景、核心观点、人话解释、机制、原片证据、工程价值和必要的最小实践。原片图均标记为原片画面并带时间戳，没有使用 AI 生成图伪装来源证据。

用户复审后，学习页进一步去除了生成适配判断和制作原则等非课程信息；每章新增两段“原讲脉络 · 中文近义转述”及章节起点时间戳，再依次呈现问题背景、核心观点、人话解释和机制。章节标题取消人为窄宽度，按正文容器自然换行。

本地入口：`local-data/visual-reading/eAXxdtNlK04/index.html`。

## 5. 与视频课堂的融合验收

- 基础入口仍是视频课堂：用户粘贴原视频链接后，先进入 `viewer.html?id=eAXxdtNlK04`。
- 图文精讲作为该课程的条件学习视图出现，不在首页建立平行内容入口。
- 课程页与图文精讲页共享 `video_id`；图文页返回时回到同一课程，而不是通用课程首页。
- 对没有足够视觉增益的访谈课程，不展示图文精讲入口。

## 6. 用户验收步骤

1. 打开本地图文页并开始计时，不先播放原视频；
2. 阅读结束后关闭页面，用自己的话回答：“为什么一个能自动改提示词的循环，仍可能只是更快地浪费 Token？”；
3. 复述应至少覆盖：目标函数信号、可修复失败、validation 泛化、领域专家和停止/逃生条件中的 4 项；
4. 随机点击 3 张图的时间戳，检查能否在原片 ±30 秒内找到对应依据；
5. 把自己项目中的一个模糊指标改写成 3 个可观察的是/否判断，并写出 fit、validation、test 的最小样本划分。

通过条件：阅读与练习前的理解时间不超过原片的 1/2；闭卷复述至少命中 4/5；3 个时间戳全部可核对；能够完成最小实践。页面观感和图片清晰度单独记录，不用低清验收素材冒充正式清晰度通过。
