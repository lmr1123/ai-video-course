# NotebookLM 基线实验：重讲视频形态验证

> 目的：回答"AI 重讲视频这个形态值不值得自建"。用托管生成的视频概览做基线，与阅读互动课程页对比学习效果。对应决策见 `docs/决策日志.md` 2026-07-12 条、方案见 `docs/重讲模式方案.md` 2.1。

## 一、生成步骤（需本人 Google 账号，约 10 分钟）

1. 打开 https://notebooklm.google.com → 新建笔记本，命名 `codex-访谈-基线`。
2. 上传本目录两个来源文件：`来源1-课程文档.md`、`来源2-原视频英文字幕.md`。
3. 右侧工作室（Studio）→ **视频概览（Video Overview）** → 点"自定义"，粘贴下面的定制提示词 → 生成（约几分钟）。
4. 顺手也生成一份**音频概览**（同样粘贴提示词），用于对照"纯音频是否已够用"。

### 定制提示词（生成时粘贴）

```text
用中文讲解。听众是有基础开发经验、会用 AI 写代码（vibe coding）的开发者，
不要解释 PRD、原型、coding agent 这类基础概念。
不要按时间顺序摘要访谈，而是按下面的课程结构重新组织：
1) 开头用一个听众真实会遇到的问题做钩子；
2) 核心模型：产品流程为什么倒转了（实现廉价 → 策展昂贵，媒介与阶段绑定断裂）；
3) 用"Codex app 11 月发布必败、2 月发布成功"和"同一功能可能要发布六次"讲清模型时机如何决定产品成败；
4) 讲清角色塌缩：角色 = 工作内容的平均位置，但为什么取消产品角色是坏主意；
5) 结尾给出听众明天就能做的一个最小实践。
引用嘉宾原话时说明这是嘉宾观点；你自己的解释要和嘉宾观点区分开。
```

## 二、测量步骤（生成后，约 40 分钟）

前置：你此前已深读过课程页，记忆会污染对照。缓解办法：本轮只测"视频概览是否把机制讲清楚"，复述以视频内容为准，不凭旧记忆补全。

1. 完整看一遍视频概览，记录时长 `T视频`。
2. 合上所有材料，写两段复述（各 3–5 句）：
   - "媒介-阶段绑定断裂"是什么、为什么发生；
   - "发布六次"背后的规划逻辑是什么。
3. 对照 `来源1-课程文档.md` 的模块一 Q4 和模块四 Q4 打分：每段复述按"关键要素齐全 / 有缺漏 / 有错误"三档。
4. 记录四个观察，写入本目录 `实验结果.md`：
   - 叙事结构：它按提示词的课程结构讲了吗，还是退化成顺序摘要？
   - 讲明白程度：两个机制哪个讲清了、哪个没有？
   - 缺失代价：没有时间戳回溯/归属标注，实际学习中你在哪一刻想要它们？
   - 音频概览对照：只听音频损失多少？

## 三、判定

- 视频概览两段复述均达"关键要素齐全"，且你主观认为"不需要再读课程页"→ 托管生成已够用，自建 L2 优先级下调，先做"课程页＋NotebookLM 导出"的轻组合。
- 复述有缺漏/错误，或明显想点回原片核对 → 自建 L2 按方案推进，实验记录反哺重讲脚本的 prompt 设计。
- 无论结果如何，把 `实验结果.md` 的结论同步到 `tasks/progress.md` 和 `docs/重讲模式方案.md` 2.1 的状态标注。
